Reading data into R


Before we store our dataset into a variable, we have to download it. Here is the open data which I took from The Word Bank.



data1=read.csv(file.choose())         # read csv file into R and store in a named variable



Print the output of ‘data1’ into R console occupies large space and it is undesired. One way to deal with it is to change the class of ‘data1’ from data frame to tibble. And this require to load the Tibble package from Tidyverse library. Tidyverse is my personal favourite and it consists list of useful analysis R packagse like ggplot2, tidyr, dyplr and so on.




nrow(data1)                            # check number of rows for variable
[1] 264

ncol(data1)                            # check number of columns for variable

library('tidyverse')                   # load library

class(data1)                           # determine the class of variable
[1] "data.frame"

data1=as.tibble(data1)                 # change class of variable and store

Data wrangling on R


As output produced is not in tidy format(one variable in one column and every observations have its row), it is difficult for us to do our analysis and visualise later. Not only that, in this case, there are some variables which you might have no interest to include it in tibble(table) like ‘Indicator.name’ and ‘Indicator.Code’. To overcome these kind of issues, dplyr and tidyr from Tidyverse library are always helpful in manipulating and tidying data.

So, we have a scenario here. Let say we want to include the name of countries and their amount of green house gasses released from 2008 to 2011 by 4 columns. First column is for the list of countries, while second column is for code of countries, third column is for years and fourth column is for total green house gasses released in kilo tonnes of carbon dioxide equivalent.



I


data1=select(data1,Country.Name,Country.Code,X2008,X2009,X2010,X2011)   

# select desired variables

data1=gather(data1,'X2008','X2009','X2010','X2011',key='Year',value='Total greenhouse gas emissions (kt of CO2 equivalent)')    

# tidy data



‘%>%’ is a pipe operator which indicates ‘and then’. Pipe operator is a convenient tool from Magrittr package but when we have Tidyverse library, it will be unnecessary for us to intall and load Magrittr manually because it’s part of the library.

By using ‘%>%’, we will get the same output with above code in one shot.



II


data1=data1 %>% select(Country.Name,Country.Code,X2008,X2009,X2010,X2011) %>% gather('X2008','X2009','X2010','X2011',key='Year',value='Total greenhouse gas emissions (kt of CO2 equivalent)')

# select() specifies which column you wish to include by column name

# gather() tidy data

Gsub function


I never found gsub function is so useful until I met this problem during that time. As you can notice that in ‘Year’ colunmn, all the entries are followed by a ‘X’ character. By using gusub function, this issue is not longer an issue anymore.




data1$Year=gsub('X','',data1$Year)               # replace 'X' character with empty
